# Learning to Communicate with Deep Multi-Agent Reinforcement Learning

This is a PyTorch implementation of the [original Lua code release](https://github.com/iassael/learning-to-communicate). 

Chào bạn, tôi là GitHub Copilot. Tôi sẽ hướng dẫn bạn chi tiết về project này.

Đây là một dự án nghiên cứu về "học giao tiếp" (learning to communicate) giữa các tác nhân (agents) sử dụng Reinforcement Learning (RL) và PyTorch. Mục tiêu là huấn luyện nhiều agents để chúng học cách phối hợp với nhau thông qua giao tiếp để giải quyết một nhiệm vụ chung.

Hãy bắt đầu với các file và khái niệm cốt lõi.

### 1. Tổng quan luồng hoạt động

Luồng hoạt động chính của project như sau:

1.  **Khởi chạy (`main.py`):** Bạn sẽ chạy file này từ terminal, cung cấp cho nó một file cấu hình (trong thư mục `config/`).
2.  **Thiết lập môi trường:** `main.py` sẽ đọc file cấu hình để biết cần chơi game gì (`switch`), có bao nhiêu agents, và các tham số cho model neural network.
3.  **Tạo Game và Agents:**
    *   Một đối tượng `SwitchGame` được tạo ra, định nghĩa luật chơi, trạng thái, và phần thưởng.
    *   Nhiều `CNetAgent` được tạo ra. Mỗi agent có một bộ não riêng (một mạng neural network `SwitchCNet`) để quyết định hành động và thông điệp giao tiếp.
4.  **Huấn luyện (`arena.py`):**
    *   Đối tượng `Arena` sẽ điều phối quá trình huấn luyện.
    *   Trong mỗi "episode" (một lượt chơi), `Arena` sẽ:
        *   Lấy trạng thái từ `SwitchGame`.
        *   Gửi trạng thái cho từng `CNetAgent`.
        *   Mỗi `CNetAgent` sẽ quyết định một hành động (ví dụ: bật công tắc) và một thông điệp để "nói" với các agents khác.
        *   `Arena` tập hợp các hành động và gửi lại cho `SwitchGame` để cập nhật trạng thái.
        *   `SwitchGame` trả về trạng thái mới và phần thưởng (reward).
        *   Các agents sẽ cập nhật "bộ não" của mình dựa trên phần thưởng nhận được, với mục tiêu là tối đa hóa phần thưởng trong tương lai. Quá trình này lặp đi lặp lại hàng ngàn lần.
5.  **Lưu kết quả:** Quá trình huấn luyện và phần thưởng của mỗi episode sẽ được lưu vào file CSV để phân tích sau.

### 2. Phân tích chi tiết các file

#### `requirements.txt`

File này liệt kê các thư viện Python cần thiết để chạy project. Các thư viện chính bao gồm:
*   `torch`: Framework chính để xây dựng và huấn luyện mạng neural network.
*   `numpy`: Thư viện cho tính toán khoa học, đặc biệt là xử lý mảng dữ liệu.
*   `matplotlib`, `pandas`: Dùng để phân tích và vẽ biểu đồ kết quả.

#### `main.py` - Điểm khởi đầu

Đây là file bạn sẽ thực thi để bắt đầu mọi thứ.

*   **Mục đích:** Đọc cấu hình, khởi tạo game, các agents, và môi trường huấn luyện (`Arena`), sau đó bắt đầu quá trình huấn luyện.
*   **Biến quan trọng:**
    *   `parser`: Sử dụng `argparse` để nhận các tham số từ dòng lệnh, quan trọng nhất là `--config_path` để chỉ định file cấu hình.
    *   `opt`: Một đối tượng `DotDic` (từ `utils/dotdic.py`) chứa tất cả các tham số từ file cấu hình. `DotDic` giúp truy cập các giá trị trong dictionary bằng dấu chấm (ví dụ: `opt.game` thay vì `opt['game']`).
*   **Hàm quan trọng:**
    *   `run_trial(opt, ...)`: Hàm chính để chạy một "phiên" huấn luyện. Nó khởi tạo mọi thứ và gọi `arena.train()`.
    *   `create_game(opt)`: Dựa vào `opt.game`, nó tạo ra một đối tượng game tương ứng (ở đây là `SwitchGame`).
    *   `create_agents(opt, game)`: Tạo ra danh sách các agents. Nó cũng tạo ra mạng neural (`cnet`) cho mỗi agent.
    *   `if __name__ == '__main__':`: Khối mã chính được thực thi khi bạn chạy `python main.py`. Nó xử lý việc đọc tham số dòng lệnh và gọi `run_trial`.

**Logic hoạt động:** Khi bạn chạy `python main.py --config_path config/switch_3_dial.json`, script sẽ:
1.  Đọc file `switch_3_dial.json`.
2.  Tạo một game "switch" với 3 agents.
3.  Tạo 3 agents, mỗi agent có một mạng neural riêng.
4.  Bắt đầu huấn luyện chúng trong `Arena`.

## Overview
This codebase implements two approaches to learning discrete communication protocols for playing collaborative games: Reinforced Inter-Agent Learning (RIAL), in which agents learn a factorized deep Q-learning policy across game actions and messages, and Differentiable Inter-Agent Learning (DIAL), in which the message vectors are directly learned by backpropagating errors through a noisy communication channel during training, and discretized to binary vectors during test time. While RIAL and DIAL share the same individual network architecture, one would expect learning to be more efficient under DIAL, which directly backpropagates downstream errors during training, a fact that is verified in comparing the performance of the two approaches.

## More info

Dưới đây là mô tả chi tiết hơn về các hàm và biến chính trong mã nguồn.

### `main.py`

Đây là file thực thi chính của chương trình.

- **`init_action_and_comm_bits(opt)`**: Hàm này khởi tạo các thiết lập liên quan đến hành động (action) và giao tiếp (communication) của các agent.
    - `opt.comm_enabled`: (Boolean) `True` nếu agent có khả năng giao tiếp (`game_comm_bits` > 0 và có nhiều hơn 1 agent).
    - `opt.model_comm_narrow`: (Boolean) Nếu là `True`, DRU (Differentiable Rounding Unit) sẽ sử dụng hàm `sigmoid` để正規化 (regularization). Ngược lại, sử dụng `softmax`.
    - `opt.game_comm_bits`: (Integer) Số bit trong mỗi tin nhắn. Nếu `model_comm_narrow` là `False`, giá trị này sẽ được tính bằng `2 ** game_comm_bits`.
    - `opt.game_action_space_total`: (Integer) Tổng không gian hành động, bao gồm cả hành động trong game và các hành động giao tiếp.

- **`init_opt(opt)`**: Khởi tạo các giá trị mặc định cho các tùy chọn cấu hình nếu chúng chưa được cung cấp.

- **`create_game(opt)`**: Tạo một đối tượng game dựa trên tên game được cung cấp trong `opt.game`.

- **`create_cnet(opt)`**: Tạo mô hình mạng nơ-ron (Communication Net) cho các agent.

- **`create_agents(opt, game)`**: Tạo danh sách các agent. Mỗi agent là một đối tượng `CNetAgent`.
    - Nếu `opt.model_know_share` là `True`, tất cả các agent sẽ chia sẻ cùng một mô hình mạng.
    - Ngược lại, mỗi agent sẽ có một mô hình mạng riêng.

- **`run_trial(opt, result_path=None, verbose=False)`**: Chạy một thử nghiệm (trial) huấn luyện.
    - `opt`: Đối tượng chứa các thông số cấu hình.
    - `result_path`: (String) Đường dẫn để lưu kết quả.
    - `verbose`: (Boolean) Nếu là `True`, in kết quả của mỗi epoch ra màn hình.

### `arena.py`

File này định nghĩa môi trường (Arena) nơi các agent tương tác với game.

- **`create_episode()`**: Tạo một "tập" (episode) mới để ghi lại quá trình chơi.
    - `episode.steps`: (Tensor) Số bước đã thực hiện trong episode cho mỗi batch.
    - `episode.ended`: (Tensor) Cờ báo hiệu episode đã kết thúc hay chưa.
    - `episode.r`: (Tensor) Tổng phần thưởng (reward) tích lũy.
    - `episode.step_records`: (List) Danh sách ghi lại thông tin của từng bước.

- **`create_step_record()`**: Tạo một bản ghi cho một bước (step) trong episode.
    - `record.s_t`: Trạng thái (state) của game tại bước `t`.
    - `record.a_t`: Hành động (action) mà các agent thực hiện.
    - `record.comm`: Tin nhắn (communication) được gửi đi.
    - `record.hidden`: Trạng thái ẩn (hidden state) của RNN cho mỗi agent.
    - `record.q_a_t`: Giá trị Q của hành động đã chọn.
    - `record.q_a_max_t`: Giá trị Q lớn nhất của hành động ở bước tiếp theo (dùng cho target network).

- **`run_episode(agents, train_mode=False)`**: Chạy một episode hoàn chỉnh.
    - Vòng lặp chính của game, nơi các agent lần lượt chọn hành động và giao tiếp.
    - Cập nhật trạng thái game, tính toán phần thưởng.
    - Nếu ở chế độ huấn luyện (`train_mode=True`) và có sử dụng target network (`model_target=True`), nó sẽ thực hiện một lượt forward pass trên target network để tính toán giá trị Q cho bước tiếp theo.

- **`train(agents, ...)`**: Vòng lặp huấn luyện chính.
    - Chạy nhiều episode để huấn luyện các agent.
    - Sau mỗi episode, gọi hàm `learn_from_episode` của agent để cập nhật trọng số của mô hình.
    - Định kỳ chạy các episode ở chế độ kiểm thử (test mode) để đánh giá hiệu năng.

### `agent.py`

File này định nghĩa lớp `CNetAgent`, đại diện cho mỗi agent trong game.

- **`__init__(self, opt, game, model, target, index)`**: Hàm khởi tạo.
    - `model`: Mô hình mạng nơ-ron chính.
    - `target`: Mô hình target network (một bản sao của mô hình chính, được cập nhật chậm hơn).
    - `dru`: Đối tượng DRU (Differentiable Rounding Unit) để xử lý tin nhắn.

- **`select_action_and_comm(self, step, q, eps=0, target=False, train_mode=False)`**: Chọn hành động và tin nhắn.
    - Sử dụng chiến lược epsilon-greedy: với xác suất `eps`, chọn một hành động/tin nhắn ngẫu nhiên; ngược lại, chọn hành động/tin nhắn có giá trị Q cao nhất.
    - `q`: (Tensor) Đầu ra của mô hình mạng, chứa các giá trị Q cho tất cả các hành động và tin nhắn.
    - Nếu là DIAL (`model_dial=True`), tin nhắn được tạo ra bằng cách đưa giá trị Q qua DRU.
    - Nếu là RIAL, tin nhắn được chọn như một hành động thông thường.

- **`episode_loss(self, episode)`**: Tính toán hàm mất mát (loss) cho một episode.
    - Sử dụng công thức Temporal Difference (TD) error.
    - `td_action = r_t + gamma * Q_max(s_{t+1}, a') - Q(s_t, a_t)`
    - Loss là tổng bình phương của các TD error.
    - Với RIAL, loss bao gồm cả TD error cho hành động và TD error cho tin nhắn.

- **`learn_from_episode(self, episode)`**: Cập nhật mô hình dựa trên loss đã tính.
    - Thực hiện quá trình lan truyền ngược (backpropagation) và cập nhật trọng số bằng thuật toán RMSprop.
    - Định kỳ cập nhật trọng số của target network bằng trọng số của mô hình chính.

More generally, `main.py` takes multiple arguments:

| Arg | Short | Description | Required? |
| ------ | ------ | ------- | ------- | 
| --config_path | -c | path to JSON configuration file | ✅ |
| --results_path | -r | path to directory in which to save results per trial (as csv) | - |
| --ntrials | -n | number of trials to run | - |
| --start_index | -s | start-index used as suffix in result filenames | - |
| --verbose | -v | prints results per training epoch to stdout if set | - |  

##### Configuration
JSON configuration files passed to `main.py` should consist of the following key-value pairs:

| Key | Description | Type |
| ------ | ------ | ------- |
| game | name of the game, e.g. "switch" | string |
| game_nagents | number of agents | int |
| game_action_space | number of valid game actions | int |
| game_comm_limited | true if only some agents can communicate at each step | bool |
| game_comm_bits | number of bits per message | int |
| game_comm_sigma | standard deviation of Gaussian noise applied by DRU | float |
| game_comm_hard | true if use hard discretization, soft approximation otherwise | bool |
| nsteps | maximum number of game steps | int |
| gamma | reward discount factor for Q-learning | float |
| model_dial | true if agents should use DIAL | bool |
| model_comm_narrow | true if DRU should use sigmoid for regularization, softmax otherwise | bool |
| model_target | true if learning should use a target Q-network | bool |
| model_bn | true if learning should use batch normalization | bool |
| model_know_share | true if agents should share parameters | bool |
| model_action_aware | true if each agent should know their last action | bool |
| model_rnn_size | dimension of rnn hidden state | int |
| bs | batch size of episodes, run in parallel per epoch | int |
| learningrate | learning rate for optimizer (RMSProp) | float |
| momentum | momentum for optimizer (RMSProp) | float |
| eps | exploration rate for epsilon-greedy exploration | float |
| nepisodes | number of epochs, each consisting of <bs> parallel episodes | int |
| step_test | perform a test episode every this many steps | int |
| step_target | update target network every this many steps | int |

##### Visualizing results
You can use `analyze_results.py` to graph results output by `main.py`. This script will plot the average results across all csv files per path specified after `-r`. Further, `-a` can take an alpha value to plot results as exponentially-weighted moving averages, and `-l` takes an optional list of labels corresponding to the paths.
```
$ python util/analyze_results -r <paths to results> -a <weight for EWMA>
```
